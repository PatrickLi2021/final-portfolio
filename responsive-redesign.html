Responsive Redesign


Revamping the Lower Providence Community Library Website

Context and Overview:


The Lower Providence Community Library is a public library located in Eagleville, PA in the suburbs of Philadelphia. The goal of this project was to evaluate this website for its accessibility, visual design, responsiveness, and usability and create a more responsive interface using HTML and CSS. Another objective was to brainstorm and complete a full cycle of the design process from sketches to implementation to gain experience with the design process from start to finish.

The original website had several flaws related to navigation, layout, accessibility, and user engagement. By redesigning the website, I aimed to enhance user satisfaction and make it easier for visitors to find information, access library resources, and engage with the community.


Research:


I began by analyzing the existing website of the Lower Providence Community Library to identify usability problems and areas for improvement. In order to classifty the issues in a more organized way, I divided them up into 4 main categories: efficiency, layout and navigation, learnability, and memorability. An overview of the issues is shown below:


Efficiency Issues


  • The user must manually click through different books to see new releases instead of searching for them.

  • The user must manually click through multiple events to find the one they're looking for instead of having them display on the home screen.

  • There are unlabelled images that lead to a lot of unnecessary scrolling and confusion.

  • There’s no easy way for the user to register for a library account directly on the homepage.


Layout and Navigation Issues


  • The navigation bar is at the bottom of the page when the user first arrives at the website.

  • The main picture at the top is blurry and should be smaller so users know what they are scrolling down to.

  • The logo for the library at the top of the website is small and doesn’t have a lot of contrast with the background.

  • On an enlarged monitor, the font of the items in the navigation bar is too small.

  • The 4 red action buttons are not the same size and have low color contrast against their background.


Learnability Issues


  • The location of the library events in a submenu item doesn't help users to remember where to locate events.

  • There is no easy way for the users to locate how to log into their account.


Memorability Issues


  • Some headings, such as “Welcome to the Library!” on the main page, are not centered.

  • There is no page layout consistency for the Adult Services, Child Services, and Teen Services pages.


Accessibility Issues


  • The main header of the webpage has low contrast against its background picture.

  • There is a sliding gallery in the main body of the web page that contains 2 pictures without any alternative text and several instances of redundant title text or small title text.

  • A lot of the text on the navigation bar or on the main body is too small for the user to easily read and there is some text that is cut off on a mobile device.

Some attributes that I wanted to include in the final sketch were having the navigation bar stick to hte top of the page, making each button the same dimensions, and include the upcoming events on the homepage. Low-Fidelity Wireframing:


Using my final sketch as a reference, I proceeded to create 3 wireframes for each screen size (mobile, tablet, and desktop). Each wireframe contained the same content and was designed in a way that made information accessible to the user in an intuitive way, even on smaller screens. Below, I have pictured the desktop low-fidelity wireframe, which contains the same components as the other ones.


Some key features of this design include:


  • Library Name: The name of the library is bolded against a faint background so that users have better memorability of the title page.

  • Logo: The logo is at the top left of the screen so that it's the first thing that users see when they load the web page.

  • Navigation Menu: The wireframes put the nav bar at the top of the screen, making locating the nav bar more intuitive than at the bottom. On a mobile device, the nav bar will be oriented vertically instead of horizontally.

  • Welcome Message: The “Welcome to the Library!” title is also bolded and centered to improve memorability and create a sense of symmetry and hierarchy.

  • Buttons: The 4 main buttons are all the same size and have enough spacing to see each line of text clearly.

  • Newest Book Arrivals: The latest books to have arrived are more neatly organized and have a search option for users to quickly look up books.

  • Reading Challenges: The reading challenges section has its widgets organized vertically and is not as spaced out as the original website.

  • Upcoming Events: Instead of putting the upcoming events on a Google Calendar in one of the menu tabs, this feature can show the most imminent upcoming events that is automatically updated and then have the rest on the calendar. On the mobile and tablet versions, they will be organized in a column to incorporate responsiveness.

Solution:


Using HTML and CSS, I developed the responsive final website, ensuring compatibility with different devices and screen sizes. The website passed accessibility checks with zero errors and zero alerts, demonstrating its improved usability and accessibility. The final website can be found at the following link: Lower Providence Community Library.



Learnings and Conclusion:


This project gave me great exposure to the iterative design process. Iterating on wireframes and mockups taught me how to identify and address usability issues effectively. Additioanlly, I gained a deeper understanding about the importance of accessibility and how to incorporate accessible features from the beginning from the beginning to create a more inclusive user experience. Lastly, I was able to practice performing responsive design using HTML and CSS. Designing for multiple devices requires careful consideration of layout, navigation, and content presentation and I was able to hone my skills in those areas.
